# StudentBuddy - Телеграм бот для студентов

Бот для получения расписания занятий колледжа с поддержкой уведомлений и замен.

## Возможности

- 📅 Просмотр расписания по дням недели
- 🔔 Уведомления за 10 минут до начала урока
- 🔄 Информация о заменах в расписании
- 🏢 Поддержка нескольких корпусов
- 💾 Надежное хранение данных в SQLite

## Функциональность бота

### Назначение

StudentBuddy — это Telegram-бот для студентов колледжа. Он показывает расписание занятий по группе и корпусу, присылает напоминания перед уроками и отображает замены. Данные берутся с внешнего API (api.mgkeit.space), настройки и группа пользователя хранятся в SQLite.

### Первый запуск и регистрация

1. **`/start`** — точка входа.
2. Если пользователь **новый** или **без группы** в базе:
   - бот приветствует по имени (из Telegram: имя, при отсутствии — username, иначе «друг»);
   - показывает список корпусов кнопками: Судостроительная, Коломенская, Академика Миллионщикова, Бирюлёво;
   - после выбора корпуса просит ввести **номер группы** в формате типа `1ИП-3-25` (регулярка: цифры + буквы + дефис + цифры + дефис + цифры);
   - группа проверяется через API расписания: если группы нет в выбранном корпусе или формат неверный — бот просит ввести снова или сменить корпус;
   - после успешной проверки корпус и группа сохраняются в БД, показывается главное меню.
3. Если пользователь **уже есть в БД с корпусом и группой** — сразу показывается главное меню без повторного выбора.

В любой момент можно нажать **«Изменить данные»** в меню — снова пройдётся шаг выбора корпуса и ввода группы (с проверкой через API). Кнопки **«Назад»** (на шаге группы) и **«❌ Отмена»** прерывают диалог или отменяют текущее действие.

### Главное меню

- **`/menu`** — выводит описание меню и кнопки под сообщением.
- Под приветствием отображаются инлайн-кнопки:
  - **📅 Расписание** — выбор дня недели (Пн–Сб или «Сегодня»), затем показ расписания на выбранный день;
  - **🔔 Уведомления** — вкл/выкл напоминаний за 10 минут до начала урока (состояние хранится в БД);
  - **Изменить данные** — смена корпуса/группы (см. выше).

Кнопка **«❌ Отмена»** вне диалога выбора корпуса/группы просто сообщает, что отменять нечего, и показывает меню.

### Расписание

- **`/schedule`** — расписание на **сегодня** (или на субботу, если сегодня воскресенье).
- **`/schedule пн`**, **`/schedule вт`**, … **`/schedule сб`** (или полные названия/алиасы: понедельник, вторник и т.д.) — расписание на выбранный день. День можно задать и числом 0–5 (0 = понедельник).
- Кнопка **«📅 Расписание»** открывает выбор дня кнопками (Пн, Вт, Ср, Чт, Пт, Сб, Сегодня). После выбора дня бот запрашивает расписание у API по сохранённым корпусу и группе и выводит отформатированное расписание; внизу добавляется блок **замен** на этот день (если API замен отдаёт данные).

Расписание и замены запрашиваются с api.mgkeit.space (Bearer-токен `SCHEDULE_API_TOKEN`). Используется текущая неделя (`week=current`). Длинные сообщения обрезаются до 4000 символов.

### Уведомления за 10 минут до урока

- Раз в минуту job-задача в боте обходит всех пользователей с **включёнными уведомлениями** в БД.
- Для каждого берётся расписание на **текущий день** по его группе и корпусу.
- Если начало какого-то урока попадает в интервал **через 9–11 минут** от текущего времени, пользователю отправляется одно напоминание (предмет, кабинет, преподаватель, время). Повторное напоминание по тому же уроку не отправляется (ключ по дате и времени урока в user_data).
- Время для сравнения берётся в **часовом поясе расписания (Europe/Moscow)** — так напоминания приходят вовремя даже если бот запущен на сервере в UTC или другом поясе.

Если JobQueue недоступен (например, при некоторых способах запуска), напоминания не работают; в логах будет предупреждение.

### Данные и API

- **Пользователи:** SQLite. Для каждого пользователя хранятся: Telegram user_id, имя, корпус, группа, включены ли уведомления, даты создания/обновления.
- **Расписание:** `timetable/api.py` — POST `api.mgkeit.space/api/v1/timetable` (группа, корпус, неделя, день). Поддерживается кэш по ETag (304).
- **Замены:** `replacements/api.py` — POST `api.mgkeit.space/api/v1/replacements` (дата, группа, опционально корпус). К выводу расписания на день добавляется блок замен для этой группы.
- **Корпуса:** в боте используется фиксированный список корпусов из `config.py`; API корпусов (`buildings/api.py`) доступен для получения списка с сервера (при необходимости можно перейти на него).

Состояние диалога (корпус/группа во время ввода) сохраняется через ConversationHandler и PicklePersistence, чтобы при перезапуске бота пользователь не терял шаг ввода.

## Установка

### 1. Клонирование репозитория

```bash
git clone <repository-url>
cd StudentBuddy
```

### 2. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 3. Настройка окружения

Создайте файл `.env` на основе `.env.example`:

```bash
cp .env.example .env
```

Отредактируйте `.env` и добавьте:
- `BOT_TOKEN` - токен вашего Telegram бота (получите у @BotFather)
- `SCHEDULE_API_TOKEN` - токен API для получения расписания

### 4. Запуск бота

```bash
python3 bot.py
```

## Деплой на хост (VPS / сервер)

Проект готов к заливке на хост. На сервере:

1. **Клонируй репозиторий** в нужную папку (например `/opt/StudentBuddy` или домашний каталог).
2. **Python 3.9+** должен быть установлен.
3. **Виртуальное окружение** (рекомендуется):
   ```bash
   python3 -m venv .venv
   source .venv/bin/activate   # Linux/macOS
   pip install -r requirements.txt
   ```
4. **Создай `.env`** из `.env.example` и заполни `BOT_TOKEN` и `SCHEDULE_API_TOKEN`.
5. **Запуск из папки проекта** — все пути к БД и pickle относительные:
   ```bash
   python3 bot.py
   ```
   Для фоновой работы: `nohup python3 bot.py &` или настрой systemd/supervisor.
6. **Бэкапы БД**: настрой cron (см. CHECKLIST.md), например раз в день:
   `0 3 * * * cd /path/to/StudentBuddy && python3 db_admin.py backup`

Файлы `.env`, `studentbuddy.db` и `studentbuddy_data.pickle` в `.gitignore` — на хост их не коммитить, создавать и хранить только на сервере.

## База данных

Бот использует SQLite3 для хранения данных пользователей. База данных автоматически создается при первом запуске.

### Структура данных

Для каждого пользователя хранится:
- ID пользователя Telegram
- Имя пользователя
- Выбранный корпус
- Номер группы
- Настройки уведомлений
- Даты создания и обновления

Подробная документация: [DATABASE.md](DATABASE.md)

### Миграция из старой версии

Если вы обновляете бот со старой версии (с pickle), выполните миграцию:

```bash
python3 migrate_to_db.py
```

### Управление базой данных

Для просмотра статистики и управления данными используйте:

```bash
# Статистика
python3 db_admin.py stats

# Список пользователей
python3 db_admin.py list

# Информация о пользователе
python3 db_admin.py user <user_id>

# Резервная копия
python3 db_admin.py backup
```

## Разработка

### Требования

- Python 3.9+
- python-telegram-bot
- httpx
- python-dotenv

### Добавление новых функций

1. Создайте обработчик в `handlers/`
2. Зарегистрируйте его в `bot.py`
3. При необходимости добавьте новые поля в БД (см. `database.py`)

### Резервное копирование

Регулярно создавайте резервные копии базы данных:

```bash
python3 db_admin.py backup
```

Или настройте автоматическое резервное копирование через cron:

```bash
# Каждый день в 3:00
0 3 * * * cd /path/to/StudentBuddy && python3 db_admin.py backup
```

## Устранение неполадок

### База данных заблокирована

Если вы видите ошибку "database is locked":
1. Убедитесь, что запущена только одна копия бота
2. Закройте все SQLite клиенты, подключенные к БД

### Не приходят уведомления

1. Проверьте, что уведомления включены (кнопка 🔔)
2. Убедитесь, что указаны корпус и группа
3. Проверьте логи бота на наличие ошибок

### Неверное расписание

1. Проверьте правильность группы и корпуса
2. Обновите данные через "Изменить данные"
3. Убедитесь, что API токен корректен
