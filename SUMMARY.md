# 📋 Резюме: База данных SQLite3 для StudentBuddy

## 🎯 Что было сделано

Реализована полноценная система хранения данных пользователей на базе SQLite3.

### ✅ Основные возможности

1. **Надежное хранение данных**
   - Группа пользователя
   - Выбранный корпус
   - Настройки уведомлений
   - Имя и контактная информация
   - История изменений (created_at, updated_at)

2. **Автоматическое обновление данных**
   - При изменении группы через "Изменить данные"
   - При переключении уведомлений
   - При смене корпуса

3. **Миграция из старой версии**
   - Скрипт для переноса данных из pickle
   - Сохранение всех существующих пользователей
   - Совместимость с предыдущей версией

## 🔑 Ключевые функции

### База данных

```python
from database import Database

db = Database()

# Добавление пользователя
db.add_user(user_id=123, username="user", name="Иван")

# Обновление группы (пользователь ошибся)
db.update_user(123, student_group="1ИП-4-25")

# Изменение корпуса
db.update_user(123, building="Коломенская")

# Управление уведомлениями
db.set_notifications(123, enabled=True)

# Получение данных
user = db.get_user(123)
print(user['student_group'])  # "1ИП-4-25"
```

### Структура таблицы users

| Поле | Тип | Описание |
|------|-----|----------|
| user_id | INTEGER | ID в Telegram (PRIMARY KEY) |
| username | TEXT | Username (@username) |
| first_name | TEXT | Имя в Telegram |
| name | TEXT | Отображаемое имя |
| building | TEXT | Корпус (например, "Судостроительная") |
| student_group | TEXT | Группа (например, "1ИП-3-25") |
| notifications_enabled | INTEGER | 1 = включены, 0 = выключены |
| created_at | TEXT | Дата создания (ISO) |
| updated_at | TEXT | Дата обновления (ISO) |

## 🎨 Сценарии использования

### 1. Новый пользователь

1. Пользователь отправляет `/start`
2. Выбирает корпус → сохраняется в БД
3. Вводит группу → сохраняется в БД
4. Данные доступны для расписания и уведомлений

### 2. Изменение группы (ошибка пользователя)

1. Пользователь нажимает "Изменить данные"
2. Выбирает новый корпус
3. Вводит правильную группу
4. **Старые данные автоматически заменяются новыми в БД**
5. Расписание теперь показывается для новой группы

### 3. Уведомления

1. Пользователь переключает уведомления (кнопка 🔔)
2. Состояние сохраняется в БД
3. JobQueue проверяет БД каждую минуту
4. Отправляет напоминания только тем, у кого включены уведомления

### 4. Перезапуск бота

1. Бот перезапускается
2. При первом запросе `/schedule` данные загружаются из БД
3. Пользователю не нужно проходить `/start` заново
