# Быстрый старт

## Обновление с предыдущей версии

Если вы обновляете бот со старой версии:

### 1. Остановите бота

```bash
# Найдите процесс бота
ps aux | grep bot.py

# Остановите его (замените PID на реальный)
kill <PID>
```

### 2. Обновите код

```bash
git pull origin main
```

### 3. Мигрируйте данные в SQLite

```bash
python3 migrate_to_db.py
```

Вы увидите:
```
Начинаем миграцию данных из pickle в SQLite...
Мигрирован пользователь 123456789: группа 1ИП-3-25
Мигрирован пользователь 987654321: группа 2ИП-1-24
==================================================
Миграция завершена:
  Мигрировано: 2
  Пропущено: 0
  Ошибок: 0
==================================================
```

### 4. Проверьте данные

```bash
python3 db_admin.py stats
```

### 5. Запустите бота

```bash
python3 bot.py
```

## Новая установка

### 1. Установите зависимости

```bash
pip install -r requirements.txt
```

### 2. Настройте .env

```bash
cp .env.example .env
nano .env  # Добавьте BOT_TOKEN и SCHEDULE_API_TOKEN
```

### 3. Запустите бота

```bash
python3 bot.py
```

База данных создастся автоматически при первом запуске.

- Интерфейс пользователя не изменился

## Ключевые возможности БД

### 1. Изменение данных

Пользователь может в любой момент:
- Изменить группу через кнопку "Изменить данные"
- Сменить корпус
- Включить/выключить уведомления

Все изменения автоматически сохраняются в БД:

```python
# Код обновляет БД автоматически при любых изменениях
db.update_user(user_id, student_group="новая группа")
```

### 2. Надежность

- Данные не теряются при сбое
- Автоматический откат транзакций при ошибках
- Защита от SQL injection

### 3. Резервное копирование

```bash
# Создать бэкап
python3 db_admin.py backup

# Результат
✅ Резервная копия создана: studentbuddy_backup_20260209_085231.db
   Размер: 12288 байт
```

## Устранение проблем

### "ModuleNotFoundError: No module named 'telegram'"

Установите зависимости:
```bash
pip install -r requirements.txt
```

### "UNIQUE constraint failed"

Это нормально при повторной миграции. Данные уже в БД.

### База данных заблокирована

Убедитесь, что запущена только одна копия бота:
```bash
ps aux | grep bot.py
```

### Старый pickle файл

После успешной миграции можете оставить pickle файл - он используется для других данных бота (context состояния, кэш и т.д.).
